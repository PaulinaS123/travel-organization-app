# Postman API Testing Documentation

## Overview

Postman was used to test and validate the Travel Organization App backend APIs connected to Supabase.

## Endpoints Tested

### Trips

- GET All Trips
- GET Single Trip
- POST Create Trip
- PATCH Update Trip
- DELETE Trip

### Authentication

- Authentication endpoints were configured and tested using API keys and bearer tokens.

## Validation Results

- Successfully retrieved records from the database.
- Successfully inserted new records.
- Successfully updated existing records.
- Successfully deleted records.
- Verified API responses and status codes.

## Evidence

See screenshots in the `screenshots` directory and the exported Postman collection in the `postman` directory.

## Outcome

The backend API endpoints function correctly and support CRUD operations required by the Travel Organization App.
